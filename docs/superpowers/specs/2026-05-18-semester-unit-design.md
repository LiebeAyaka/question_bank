# 学期+单元组合标记功能设计

## 概述

将题目的单元标记（`unit`）从单一文本输入改为学期和单元的组合选择模式，同时保留自定义输入的灵活性。

## 界面设计

### 字段设计

| 字段 | 默认模式 | 自定义模式 |
|------|----------|------------|
| 学期 | 下拉选择 | 文本输入 |
| 单元 | 下拉选择 | 文本输入 |

**学期预置选项：**
- 高一上、高一下
- 高二上、高二下
- 高三上、高三下

**单元预置选项：**
- Unidad 1 ~ Unidad 16

### 交互行为

1. **选择预置选项**：用户从下拉列表直接选择
2. **切换自定义模式**：选择"✏️ 自定义..."选项，输入框替换下拉框
3. **恢复选择**：自定义模式下，显示"↩ 恢复选择"按钮，点击返回下拉模式
4. **组合生成**：两个字段值用 `-` 连接，如 `高一上-Unidad 1`

### UI 预览

```
┌─────────────────────────────────────────┐
│ 学期              │ 单元                 │
│ [高一上        ▼] │ [Unidad 1        ▼]  │
│ 单元标记：高一年级上-Unidad 1             │
└─────────────────────────────────────────┘

自定义模式下：
┌─────────────────────────────────────────┐
│ 学期              │ 单元                 │
│ [_____________] ↩ │ [_____________] ↩    │
│ 单元标记：2024秋-期中复习                 │
└─────────────────────────────────────────┘
```

## 数据存储

- **存储格式**：单一 `unit` 字段，格式为 `{学期}-{单元}`
- **兼容性**：已有数据保持不变，编辑时正常显示

## 涉及文件

| 文件 | 修改内容 |
|------|----------|
| `src/components/QuestionForm.tsx` | 学期/单元字段改为双选择器 |
| `src/components/QuestionForm.module.css` | 样式调整（输入框+恢复按钮） |

## 状态管理

```typescript
interface UnitFormState {
  semester: string;        // 学期值（预置或自定义）
  unit: string;           // 单元值（预置或自定义）
  semesterCustomMode: boolean;  // 学期是否为自定义模式
  unitCustomMode: boolean;     // 单元是否为自定义模式
}
```

## 实现要点

1. **双向切换**：下拉 ↔ 自定义 模式可自由切换
2. **恢复机制**：自定义模式下提供"恢复选择"按钮
3. **实时预览**：底部显示最终组合的单元标记
4. **编辑兼容**：编辑已有题目时正确解析并显示现有值
